# 保存目标下载速度小数精度

## Goal

让设置页面的“目标下载速度（MiB/s）”真正支持一位小数，用户输入或使用步进控件选择的值应按 `0.1` 精度保存，并在保存后及重新加载设置时保持不变。

## Background

- 设置页当前为目标下载速度配置了 `min=0.1`、`max=5` 和 `step=0.1`，但数字输入组件没有显式声明小数精度，实际保存结果表现为整数。
- 前端状态、Wails 请求/响应、应用适配层和 core 配置字段均使用数值或 `float64`，现有配置加载测试也能读取 `2.5`，因此数据契约无需变更。
- 目标下载速度的既有有效范围是 `0.1` 至 `5.0` MiB/s。

## Requirements

- 目标下载速度输入框必须允许并保留一位小数，步进单位为 `0.1`。
- 自动保存必须将该一位小数值原样传递并持久化，不得取整。
- 保存响应和后续重新加载必须恢复相同的一位小数值。
- 保持既有最小值 `0.1`、最大值 `5.0`、默认值 `1.0` 及超范围钳制行为不变。
- 不修改 Wails 公共 API 字段名或数据类型。

## Acceptance Criteria

- [x] 在设置页输入或步进到 `1.1` 时，输入框保持显示 `1.1`，自动保存请求使用数值 `1.1`。
- [x] 保存 `1.1` 后，返回的设置视图与配置文件中的 `target_download_rate_mib` 均保留 `1.1`。
- [x] 重新加载设置后，目标下载速度仍为 `1.1`，不会变为 `1`。
- [x] `0.1` 和 `5.0` 的既有范围限制继续生效，其他网络设置不受影响。
- [x] 前端 lint/build 和相关 Go 测试通过。

## Out Of Scope

- 改变目标下载速度的单位、范围或默认值。
- 改变其他整数网络设置的输入精度。
- 修改下载器的自适应并发算法。
