# Connector compatibility

## Goal

让安装了 Connector（mod id 为 `connector`）的本地 Minecraft 实例能够在应用内按真实 Forge/NeoForge 宿主与虚拟 Fabric 两种加载器视角使用现有模组浏览、下载和本地管理能力，同时清楚地区分当前加载器不兼容的本地模组，避免它们干扰常用操作。

## Background

- Connector 通过在一个实例内兼容另一加载器生态，使单一实例可能需要同时匹配两类在线模组版本。
- 当前应用以一个 Minecraft 版本/加载器组合驱动大多数模组功能，因此需要在不重写各功能筛选逻辑的前提下，为 Connector 实例提供可切换的加载器视角。
- 本地模组管理仍需显示与当前视角不兼容的模组，但应降低它们在默认列表中的优先级和视觉干扰。
- Connector 官方项目说明其用途是让 Fabric 模组运行在 NeoForge 上，项目发布加载器范围同时包含 Forge 与 NeoForge；因此兼容组合为 Forge/NeoForge 宿主与 Fabric 虚拟视角。
- 当前后端下载路径会用全局所选实例的 `MinecraftVersion` / `ModLoader` 覆盖前端请求（`core/modbridge/modbridge.go:185`），因此切换状态必须进入后端所选版本契约，不能只在 Vue 中改筛选字段。
- 当前本地 JAR 解析只执行所选加载器对应的解析器（`core/minecraft/modparser.go:622`），而解析缓存仅以 SHA1 为键（`core/global/jarcache.go:14`）；若不调整该契约，先后切换加载器会复用错误解析结果。
- `VersionInfo.ID` 同时承担实例唯一键和磁盘目录定位（`core/global/global.go:81`、`core/minecraft/prism.go:196`），虚拟视角必须保留同一物理实例身份与路径，不能伪造第二个磁盘目录。

## Requirements

- R1. 应以本地已安装且启用的模组中存在 mod id `connector` 作为 Connector 兼容模式的触发条件；仅存在 `.jar.disabled` 文件不得触发。
- R2. 当已安装 `connector` 的真实实例加载器为 Forge 或 NeoForge 时，应用应为同一个 Minecraft 游戏版本提供 Fabric 虚拟视角，使现有依赖所选版本/加载器的功能能够复用该视角工作。
- R3. UI 应在 Connector 实例上显示明确的切换按钮，点击后在真实版本项与虚拟版本项之间切换，并可再次点击切回。
- R4. 切换必须保留同一 Minecraft 游戏版本，只改变用于筛选和匹配的加载器视角。
- R5. 不含 `connector` 的实例不得出现切换按钮或虚拟版本项，既有行为保持不变。
- R6. 本地模组管理应根据当前加载器视角识别模组兼容性，将不兼容加载器的模组统一排列到列表末尾。
- R7. 列表末尾的不兼容模组区域默认折叠，并提供显式交互让用户展开和再次折叠。
- R8. 展开后的不兼容模组仍应保留本地管理页现有的适用操作和状态展示，除非现有操作本身有独立限制。
- R9. 模组加载器识别应覆盖项目当前支持的 `fabric.mod.json`、`META-INF/mods.toml` 与 `META-INF/neoforge.mods.toml` 元数据来源；同一 JAR 声明多种元数据时应聚合为多加载器兼容，缺失或无法识别时不得武断判为不兼容。
- R10. Connector 的虚拟版本不得被误当作额外的磁盘实例，也不得造成重复扫描、重复持久化或破坏真实版本信息。
- R11. 切到 Fabric 视角时，Connector 自身及其它明确仅属于 Forge/NeoForge 的模组进入不兼容分组；切回宿主视角时，明确仅属于 Fabric 的模组进入该分组。多加载器及无法确认归属的模组留在主列表。
- R12. 虚拟 Fabric 视角为非持久化会话状态，仅在当前实例选择期间保留；重新选择实例、刷新版本或重启应用时恢复实例真实的 Forge/NeoForge 宿主加载器。
- R13. Connector 被禁用或从实例移除并刷新后，兼容模式和切换按钮应立即消失；若当前处于虚拟 Fabric 视角，应自动恢复真实宿主加载器。

## Acceptance Criteria

- [x] AC1. 在一个已识别到 `connector` 的受支持实例中，版本相关界面出现加载器切换按钮，且可在同一游戏版本的两个加载器视角间往返切换。
- [x] AC2. 切换后，依赖当前所选版本加载器的模组搜索、版本筛选和下载流程使用切换后的加载器视角；切回后恢复真实视角。
- [x] AC3. 对不含 `connector` 的实例，版本列表、选择状态和相关功能与改动前一致。
- [x] AC4. 本地模组管理默认先显示兼容、多加载器或无法确认归属的模组，并在列表末尾显示一个默认折叠的不兼容模组分组。
- [x] AC5. 用户展开不兼容分组后可查看其中全部模组并执行现有可用操作；折叠后列表恢复紧凑状态。
- [x] AC6. 多加载器模组不会被错误放入不兼容分组；仅明确判定与当前加载器视角不兼容的模组进入该分组。
- [x] AC7. 真实版本项、虚拟版本项和 Connector 检测具有自动化测试，前后端构建、lint 及相关 Go 测试通过。
- [x] AC8. 切换到 Fabric 视角后重新选择实例、刷新版本或重启应用，所选加载器恢复为磁盘识别出的真实宿主加载器，且未写入新的持久化设置。
- [x] AC9. 仅有禁用的 `connector` 模组时不显示切换按钮；启用后刷新则显示，随后再次禁用或移除并刷新会隐藏按钮并恢复真实宿主视角。

## Out of Scope

- 改变 Connector 本身的安装、更新或运行行为。
- 为任意加载器组合提供通用的跨加载器模拟；本任务只处理 Connector 实际支持且项目能够可靠识别的组合。
- 自动禁用、移动或删除被判定为不兼容的本地模组文件。

## Notes

- 本任务是跨 core、Wails adapter 与前端的复杂任务，开始实现前必须补齐 `design.md` 和 `implement.md`。
