# 修复收藏夹迁移不可见

## Goal

修复收藏夹“跨版本复制”提示成功但用户看不到结果的跨层回归，确保复制后的目标收藏夹和收藏项在目标 Minecraft 版本及 Mod Loader 作用域中可见、可继续操作，并且成功提示准确反映实际可达结果。

## Background

- 迁移弹窗当前将 `targetListId` 固定为源收藏夹 ID：`frontend/src/views/Favorites.vue:572-579`。
- 后端迁移会保留源记录，并将匹配到的目标版本作为新作用域记录 upsert 到目标收藏夹：`core/appcore/service.go:448-492`。
- `applied` 只反映数据库新增或更新，不代表结果能在当前 UI 作用域访问：`core/appcore/service.go:492`。
- 前端应用迁移后仍以当前旧作用域重新加载条目，目标作用域记录会被过滤：`frontend/src/stores/favorites.ts:62-71,339-350`。
- 作用域收藏夹本身也按创建时的 Minecraft 版本和 Mod Loader 过滤；切换到迁移目标作用域后，绑定旧作用域的原收藏夹会整体不可见：`frontend/src/stores/favorites.ts:77-82,99-107`。
- 后端迁移单元测试验证写入结果，但未覆盖迁移后收藏夹和记录的 UI 可达性。
- 产品语义确定为跨版本复制：保留源收藏夹与源记录，自动创建新的目标作用域收藏夹承载复制结果，不做原地作用域迁移或复用。
- 应用尚未发布，用户文案及 Wails、appcore、frontend store 内部契约统一从 Migration 重命名为 CrossVersionCopy，不保留旧兼容别名。
- 目标收藏夹由系统自动创建，名称与源收藏夹相同；不允许用户选择或手工命名目标收藏夹。
- 如果目标作用域已经存在同名收藏夹，不复用、不覆盖；弹窗显示明确提示并禁用执行按钮。后端应用阶段必须再次校验以防竞态绕过。

## Requirements

- R1. 跨版本复制必须自动创建一个与源收藏夹同名、且绑定目标 Minecraft 版本和 Mod Loader 的新收藏夹；复制结果写入该收藏夹并在目标作用域可见。
- R2. 必须保留源收藏夹及其原作用域记录；操作不得原地修改源收藏夹作用域。
- R3. 跨版本复制预览与应用必须使用同一目标名称和目标作用域契约；目标在预览后发生重名时不得静默写入错误位置。
- R4. 前端只能在后端写入成功且结果具备可访问目标时提示成功，并展示 added、updated、skipped 等有意义的结果。
- R5. 跨版本复制成功后关闭操作弹窗并显示明确的复制成功通知；不自动切换全局 Minecraft 版本、Mod Loader 或已选实例。用户后续手动切换到目标作用域时必须能看到新收藏夹。
- R6. 冲突处理保持现有安全行为：未忽略冲突时不写入；忽略冲突时只写入可解析项并准确报告 skipped。
- R7. 数据库 schema 提升到 v3，并在 v2 升级时清空收藏夹、收藏项、引用和收藏夹分组数据，不迁移或恢复旧收藏夹数据；固定版本、使用统计及其他无关数据必须保留。
- R8. 添加 storage 与 appcore 自动化回归测试，至少包含 scoped source、目标作用域、部分冲突、重复应用和成功后可达性；前端仓库未配置测试 runner，本任务通过 lint、类型检查/构建和完整状态流审查验证 UI，不引入新测试框架。
- R9. 收藏夹菜单、操作按钮、弹窗标题、成功/失败文案等所有用户可见位置统一使用“跨版本复制”及对应英文 “Copy Across Versions”，不再显示“迁移/Migration”。
- R10. 预览阶段检测目标作用域同名收藏夹；存在重名时展示明确冲突、禁用执行按钮。应用阶段必须原子地或通过等价保护再次检测重名，禁止复用、覆盖或创建第二个重名目标。
- R11. 自动创建目标收藏夹或写入复制内容任一步骤失败时，不得留下空的或部分写入却提示成功的目标收藏夹；结果必须可回滚或具备等价的事务一致性。

## Acceptance Criteria

- [ ] 从一个绑定旧作用域的收藏夹跨版本复制到新作用域后，源收藏夹保持不变，目标收藏夹在新作用域可见，复制成功的条目均可见且版本 ID 正确。
- [ ] 跨版本复制后不存在只能在数据库中查询、但无法通过收藏夹 UI 访问的目标记录。
- [ ] 成功反馈包含真实的 added 和 skipped 结果；零写入或目标不可达不会显示复制成功。
- [ ] 未忽略冲突时保持全量不写入；忽略冲突时仅跳过冲突项。
- [ ] 重复执行同一目标作用域复制时因目标重名被阻止，不会制造重复收藏夹或重复记录。
- [ ] 目标作用域存在同名收藏夹时，预览明确提示重名，执行按钮不可用，后端直接调用也不会写入或复用该收藏夹。
- [ ] 预览完成后若目标作用域被并发创建同名收藏夹，应用会失败且不会产生部分数据。
- [ ] 目标收藏夹创建或内容写入失败时不会遗留空收藏夹或部分复制结果。
- [ ] v2 数据库升级到 v3 后，所有旧收藏夹相关数据被清空，固定版本与使用统计保持不变，schema v3 标记正确写入。
- [ ] `go test ./...` 在 `core/` 通过。
- [ ] `npm run lint` 和 `npm run build` 在 `frontend/` 通过。
- [ ] 应用层 `go test ./...` 与 `go build ./...` 通过；若 Wails 公共契约改变，生成绑定与前端类型保持同步。
- [ ] 中英文界面不再出现将该功能称为“迁移/Migration”的用户可见文案。
- [ ] 复制成功后仅关闭操作弹窗并显示成功通知，不改变全局 Minecraft 版本、Mod Loader 或已选实例。

## Out Of Scope

- 不改变 Mod 版本匹配排序算法。
- 不重做收藏夹页面整体布局或无关的复制、引用、导出功能。
- 不提供旧收藏夹数据导入、恢复或手工修复工具。
