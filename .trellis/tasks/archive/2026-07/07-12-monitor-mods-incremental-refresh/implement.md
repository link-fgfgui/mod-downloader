# 实施计划

1. 阅读并遵循 app backend/frontend 规范，盘点 `global` 本地索引 API 与现有事件生命周期。
2. 增加文件 watcher 依赖/封装，补充 Service watcher 生命周期、实例切换和关闭清理。
3. 扩展本地索引以支持按路径移除、替换和增量扫描；抽取单文件 hash/解析逻辑并保留全量扫描行为。
4. 实现事件 debounce、事件分类、写入完成判断和失败全量兜底。
5. 修改启用/禁用/删除操作使用增量更新并发出选中版本变化事件。
6. 接入前端 Store/Manage 页面，验证自动更新和监听停止状态。
7. 添加 Go 单元测试：单文件新增/删除/重命名、重复事件合并、未变化文件不重算、实例切换隔离、watcher 关闭；运行现有前端检查与 Go 测试。
8. 使用 `trellis-check` 做规格、类型、测试和跨层数据流检查。
